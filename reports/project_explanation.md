# Project Explanation: Enterprise Data Governance & Database Analytics Command Center

## Objective

The objective of this project is to build an advanced data governance and database analytics dashboard that monitors data quality, database health, sensitive data classification, and governance issues.

## Dataset

The dataset is generated using Python and Faker and contains:

- Data catalog metadata
- Data quality check records
- Sensitive data field details
- Governance issue tracking data

## Database Tables

- data_catalog
- data_quality_checks
- sensitive_data_fields
- governance_issues

## SQL Concepts Used

- Relational database design
- Primary keys and foreign keys
- SQL joins
- Aggregations
- CASE logic
- Data quality KPI calculations
- Compliance analysis
- Governance issue reporting

## Dashboard Features

- Total tables and records monitored
- Average data quality score
- Missing record tracking
- Duplicate record monitoring
- Critical governance issue tracking
- Department-wise quality score
- Data classification analysis
- Sensitive data distribution
- Encryption status monitoring
- Governance issue severity analysis
- Data catalog table
- Downloadable data quality report

## Key Learning Outcomes

Through this project, I learned:

- Data governance concepts
- Data catalog design
- Data quality monitoring
- Sensitive data classification
- Compliance tracking
- SQL-based governance reporting
- Streamlit dashboard development
- Business KPI creation

## Conclusion

This project demonstrates how SQL and dashboard analytics can support enterprise data governance by improving data quality, visibility, ownership, compliance monitoring, and risk identification.
